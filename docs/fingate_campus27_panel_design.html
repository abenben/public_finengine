全体ナラティブ

「お金がソフトウェアになる。AIエージェントが支払う。銀行口座が複数の民間主体によって設計される。これは遠い未来の話ではなく、今すでに始まっている。では、この変化の中で、自分の会社は何を担うのか。帰ったら月曜日に、一つ問いを立ててほしい。」
  • 哲学的な問いで始めて、事業判断で終わる（T1→T2→T3→T4の流れ）
  • 成功事例の紹介合戦を避ける（T3の「壊れたとき」がブレーキ役）
  • 持ち帰りは「制度への知識」ではなく「自社は何を担うのかという問い」

オープニング（4分）19:40〜19:44

目的: 「法律の話から金融の前提を問い直す話へ」という明確な転換

藤井さんの冒頭フレーム（読み上げ用）

「関口先生、30分間ありがとうございました。法律の枠組みはよくわかりました。さて、ここからは一段引いた話をします。ステーブルコインを暗号資産として見るのではなく、金融インフラとして見る。そうすると今まで見えなかった問いが浮かびます——お金がソフトウェアになると、誰が顧客で、誰が損をして、誰がどのレイヤーを取るのか。今日のパネルはこの問いを、実際に動いている当事者たちと話す55分にしたいと思います。では、永吉さん・中井さん、今の一言ずつお願いします。」
  • 永吉さん: みんなの銀行として今「デジタルアセット関連でもっとも動いていること」を一言
  • 中井さん: SMFGデジタル戦略部として「今週取り組んでいること」を一言
  • 関口先生: スキップ（基調講演で十分に自己紹介済み）

T1: 顧客は誰になるのか（10分）19:44〜19:54

T1 顧客は誰になるのか AIエージェントが支払う時代
根本の問い

AIエージェントやAPIが支払う時代、人間の意思決定はどこに残るのか

主担当

藤井（CAO視点・自ら語る） 中心。永吉 中井 に銀行側の受け止めを聞く。関口先生: このテーマでは聴衆として聞いていればOK

進行（1分）藤井さん自ら語る
「最初のテーマ。5月12日にCircle Agent Stackが発表されました。AIエージェントがUSDCでナノ決済をする時代が現実になっています。みずほのCAOとして、私はこれをこう見ています——（自身の見方を1分で語る）——では、銀行として顧客の定義が変わるとしたら何が変わるか。永吉さんから聞きます。」
進行（3.5分）→ 永吉さんへ
「みんなの銀行の頭取として、AIエージェントが銀行口座を持ち、自律的に決済する世界を想像したとき——KYCは誰を審査するのですか？与信は誰に対して行うのですか？銀行法上の"顧客"は誰になりますか？」
進行（3.5分）→ 中井さんへ
「中井さん、SMFGのデジタル戦略部として、人ではなくソフトウェアが決済する時代、メガバンクの顧客定義はどう変わりますか？それとも変えないという判断もありますか？」
深掘り（余れば）
「人間の意思決定が金融のどこに残ると思いますか？10秒で。」（全員一言）
持ち帰り

AIが支払う時代の金融機関の顧客再定義という問い

T2: 発行・使う・つなぐ（10分）19:54〜20:04

T2 発行・使う・つなぐ 銀行のレイヤー選択
根本の問い

発行者・接続者・顧客接点のうち、自社はどのレイヤーを取るべきか

主担当

永吉 中井 中心。藤井 がレイヤー整理役。関口先生: 法的リスクを補足（2分以内）

フレーム設定（1分）
「次のテーマ。銀行のポジションを3択で考えます。発行する・使う・つなぐ。どれが正解かではなく、自社はどれを取るべきかを考えていただきたい。永吉さんから。」
→ 永吉さんへ（3分）
「みんなの銀行として、この3択で今考えているポジションはどれですか？今言える範囲で正直に。」
→ 中井さんへ（3分）
「SMFGとして、どのレイヤーを本命と見ていますか？Progmatへの参画は"つなぐ"のか"発行する"のか、どちらに近いですか？」
→ 関口先生へ（2分・補足）
「法律家として見ると、この3つのレイヤーで最も法的リスクが集中するのはどこですか？今日の基調講演の補足として教えてください。」
熱量ポイント
「今、デジタルバンクとメガバンクで全く違う答えが出ました。同じ"銀行"でも取るレイヤーが違う。これが今日一番の面白さかもしれません。」
深掘り（余れば）
「トークン化預金とステーブルコインのどちらが将来的に主流になるか——どちらに賭けますか？10秒で。」
持ち帰り

自社はどのレイヤーを取るかという経営判断の問い

T3: 壊れたとき、誰が損をするのか（10分）20:04〜20:14

T3 壊れたとき、誰が損をするのか 失敗前提の設計思考
根本の問い

デペグ、カストディアン破綻、オペレーション事故を前提にすると設計はどう変わるか

主担当

関口先生（主役） が法的・顧客保護の補助線を出す。狙い: 成功事例の紹介合戦を避け、リスクと設計論で議論の格を上げる

フレーム設定（1分）
「次のテーマは少し違います。Terra/Lunaのデペグ、FTXのカストディアン破綻。壊れたとき何が起きるかを一度前提にして考えます。関口先生から。」
→ 関口先生へ（3.5分・このテーマの主役）
「ステーブルコインがデペグした場合、日本の現行法では誰が責任を持ちますか？顧客が損をしたとき、発行者・取り扱い業者・銀行のどこに請求できますか？今日の基調講演の延長として、実務家として教えてください。」
→ 永吉さんへ（2.5分）
「デペグリスクを前提にすると、みんなの銀行としてステーブルコインを使う・つなぐ設計はどう変わりますか？」
→ 中井さんへ（2.5分）
「SMFGのリスク管理観点から見ると、どの壊れ方が最も怖いですか？デペグか、カストディアン破綻か、オペレーションか。」
注意事項
関口先生の話が長くなりそうな場合 → 「ありがとうございます。永吉さん、今の法律の話を聞いて実務上どう受け止めますか？」で次へ
持ち帰り

失敗前提の設計思考。壊れたとき誰が損をするかという問い

T4: 12ヶ月以内に動くのはどこか（8分）20:23〜20:31

T4 12ヶ月以内に動くのはどこか 全員一人称で予測
根本の問い

法人決済、AI決済、越境送金、地域経済のどこに最初の実需が出るか

フレーム（0.5分）
「最後のテーマ。12ヶ月以内という具体的な時間軸で話してください。来年の今頃、実際に動いているユースケースはどこに出ていますか？一人称でお願いします。」
順番（各2分）
  • 永吉さん — 「12ヶ月以内に『これは動く』と確信しているユースケースを一つ、一人称で」
  • 中井さん — 「SMFGとして12ヶ月以内に本番化すると確信しているものを一つ」
  • 関口先生 — 「法律家として、12ヶ月以内に法的整備が追いつく領域・追いつかない領域を一言で」（1.5分）
  • 藤井さん — 「みずほCAOとして、AI×デジタルアセットで12ヶ月以内に動くと見ているのはここです」
持ち帰り

12ヶ月以内に動く領域のリスト——自社の優先度設計に使える

まとめ（4分）20:31〜20:35

藤井さんのラップアップ（1.5分）

「今日は4つ問いを立てました。顧客は誰か。どのレイヤーを取るか。壊れたとき誰が損をするか。12ヶ月以内に何が動くか。帰ったら一つだけ、自分の組織について問いを立ててください。制度の話ではなく、自社の担い方の話です。今日のパネリストに改めて拍手をお願いします。」

逆順コメント（各30秒）

中井さん → 永吉さん → 関口先生 → 藤井さん（最後の一言）

「月曜日に何を一つ決めますか？30秒でどうぞ。」

モデレーターガイド（藤井さん向け）

最重要: 基調講演が終わった直後に必ず言う → 「ステーブルコインを暗号資産ではなく金融インフラとして見る。この前提でここから話します。」これを言わないと参加者も登壇者も「法律の続き」になる。

関口先生の時間管理（重要）

  • T1: 発言させない（聴衆として聞く）
  • T2: 補足として2分以内
  • T3: 主役として3.5分。それ以上は「ありがとうございます」で切る
  • Slido: 法律系は「補足があれば」と短く
  • T4: 1.5分以内

場が静まった時のリカバリー

  • 答えが短すぎた → 「もう少し具体的に、自社でどんな議論が起きていますか？」
  • 「わからない」と言われた → 「それが正直なところですね。では別の角度で——」
  • 話が長くなった → 「ありがとうございます。一点だけ確認させてください——」

禁止事項

  • 特定銘柄・商品への投資推奨
  • 断定的な法解釈（「〜は必ず違法です」等）
  • 競合他社の内情・非公開情報
  • 「〜についてご説明ください」型の授業形式

短縮・延長プラン

状況対応効果
時間が押したT4を「各自1分で一言のみ」に圧縮−3分
時間が押したまとめを藤井さん1分のみに−2分
時間が余ったSlido拡大「もう1問」＋2分
時間が余ったT2深掘り「トークン化預金 vs ステーブルコイン」＋2分
